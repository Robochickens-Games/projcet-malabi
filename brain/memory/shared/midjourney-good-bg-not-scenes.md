# Midjourney is for backgrounds, not accurate scenes

**The learning:** Midjourney produces **great backgrounds and atmosphere** — the
painterly, mood-setting layers it's strong at. But it's **unreliable for accurate,
specific scenes and subjects**: when we need a precise creature, device, or
composition that has to be *correct* (not just pretty), Midjourney drifts off-brief.
That collides with our bar — see [[art-scientific-realism]] / [[scientific-realism-rule]]
and [[dino-accuracy-rulings]] (subjects must be real and accurate, not approximate).

**What we'll do:** Keep Midjourney for **backgrounds / ambience**, and **find a better
tool for accurate assets** — evaluate **GPT image generation (or another tool)** for the
precise, subject-correct pieces. This is the next step in the still-open hunt for the
right asset pipeline.

**Why it matters:** Consistent with the running thread — a great, *correct* visual
result beats a cheap-but-wrong one ([[blender-asset-experiment-flop]],
[0004](../../decisions/0004-drop-local-comfyui-engine.md)). We're fine spending for the
right output; the open question was never cost, it's **which tool hits the accuracy bar**.
Midjourney answers that for backgrounds, not for scenes.
