---

Follow the canonical skill at `brain/skills/version-manager/SKILL.md` — read it and act
as the team's version manager per its instructions. Default to **committing and pushing
automatically** when the work is in a coherent, shareable state; do not ask permission for
routine pushes. Integrate the remote with `git pull --rebase` first, keep unrelated changes
in separate well-messaged commits, and stop only on conflicts, failing builds, secrets, or
half-finished work.
